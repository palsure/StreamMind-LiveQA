\begin{abstract}
Current video-language models assume the full recording is available before answering a question---they cannot operate on a live camera feed.
We present \ours, a streaming system for interactive QA on video that is still in progress.
Its Semantic Keyframe Memory (SKM) retains at most $N$ frames scored by visual novelty and temporal coverage.
A Temporal Query Router (TQR) classifies each question as \emph{instantaneous}, \emph{recent}, or \emph{historical} and passes the matching time slice to a two-stage generator: BLIP extracts captions and visual answers, Flan-T5 fuses them into a response.
We introduce \emph{LiveQA-Bench}, a benchmark of 55 diverse video streams with per-question scope labels---the first VideoQA benchmark enforcing causal access with temporal scope annotations.
On an A100 GPU, \ours achieves 54.8\% overall accuracy at 254\,ms average latency (historical 77.1\%, instant 46.2\%, recent 43.2\%).
Ablations validate each component: removing TQR drops accuracy 3.9 points, replacing SKM with FIFO costs 1.2 points, and compact memory ($N\!=\!16$) achieves the best accuracy (57.2\%).
Code, benchmark, and a live webcam demo will be released.
\end{abstract}
