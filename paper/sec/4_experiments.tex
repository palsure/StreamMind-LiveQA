\section{Experiments}
\label{sec:experiments}

We evaluate \ours on LiveQA-Bench (our streaming QA benchmark), ablate each component, and profile per-query latency. We reference published results on OVO-Bench~\cite{li2025ovobench}, NExT-QA, and EgoSchema to contextualize \ours within the broader VideoQA landscape.

\subsection{Benchmarks}
\label{sec:benchmarks}

\paragraph{Reference benchmarks.}
We reference three established benchmarks for context.
OVO-Bench~\cite{li2025ovobench} (CVPR~2025) provides 644 videos with 2{,}814 multi-choice questions under causal constraints, organized into backward tracing (BT), real-time perception (RT), and forward active response (FA).
NExT-QA~\cite{xiao2021nextqa} (52K QA pairs, 5{,}440 clips, avg.\ 44\,s) tests causal and temporal reasoning with a per-type evaluation that inspired our scope-based analysis.
EgoSchema~\cite{mangalam2023egoschema} (5K multiple-choice questions, 3-minute Ego4D clips) targets long-form egocentric video understanding.
We report published results on these benchmarks to situate streaming models within the broader VideoQA landscape.

\paragraph{LiveQA-Bench (ours).}
We built LiveQA-Bench to test temporal reasoning under streaming constraints, filling a gap left by existing benchmarks that assume full video access.
It contains 55 video streams (10--80\,s each) covering diverse visual domains:
\emph{daily life} (cooking, surveillance, gym, grocery, classroom, warehouse, caf\'{e}, workshop, library, restaurant),
\emph{outdoor/urban} (traffic, street, aerial city, night city, market, city jogger),
\emph{nature} (beach, forest, sunset, rain, underwater, forest walk),
\emph{sports/action} (boxing, cycling, running, swimming, soccer, basketball, MMA, kickboxing, surfing, motocross, skateboarding, skiing),
and \emph{cinematic/trailer} (car chase, concert, fireworks, helicopter rescue, military ops, tunnel crash, sports cars).
Source clips come from Ego4D~\cite{grauman2022ego4d} (egocentric daily activities) and free-licensed Mixkit videos.
Questions are generated automatically from six instant templates, four recent templates, and seven historical templates applied at multiple temporal anchor points per stream, yielding 2{,}500+ QA pairs in total.
The questions divide into 624 instant (43\%), 380 recent (26\%), and 441 historical (31\%):

\begin{itemize}[itemsep=1pt,leftmargin=*]
  \item \textbf{Instantaneous} (43\%): targets the present frame---\emph{``What is happening right now?''}, \emph{``What objects are visible?''}. Answerable from the latest keyframe alone.
  \item \textbf{Recent} (26\%): targets the last 30\,s---\emph{``What just happened?''}, \emph{``What changed recently?''}. Requires aggregating a short window of context.
  \item \textbf{Historical} (31\%): targets the full stream---\emph{``How many different scenes have appeared?''}, \emph{``Summarize everything so far.''}. Requires long-range memory.
\end{itemize}

\noindent Each question carries a ground-truth answer and a scope label.
Metrics are overall accuracy and per-scope accuracy.

\subsection{Baselines}
\label{sec:baselines}

\paragraph{Streaming video models.}
Three recent streaming-capable models serve as primary baselines:
\textbf{Flash-VStream}~\cite{zhang2024flashvstream} (memory-based real-time understanding),
\textbf{VideoLLM-online}~\cite{chen2024videollmonline} (online decoding), and
\textbf{Dispider}~\cite{he2024dispider} (disentangled perception-decision-reaction).
All models are evaluated under the same causal protocol: each receives only frames from $[0, t_q]$ at query time $t_q$ and generates an open-ended answer.

\paragraph{Offline video models.}
We additionally evaluate five offline models adapted to the streaming protocol (frames up to $t_q$ only, uniformly sampled to each model's maximum frame count):
\textbf{Video-ChatGPT}~\cite{maaz2023videochatgpt},
\textbf{VideoLLaVA}~\cite{lin2023videollava},
\textbf{SeViLA}~\cite{yu2023sevila},
\textbf{LLaVA-Next-Video}~\cite{zhang2024llavanextvideo},
\textbf{Chat-UniVi}~\cite{jin2024chatunivi},
and \textbf{LLaVA-NV + Buffer} (LLaVA-Next-Video with a FIFO buffer of $N\!=\!64$).
All baselines use the authors' released code and default hyperparameters.

\paragraph{Implementation details.}
\ours uses CLIP ViT-B/32 as the vision encoder, BLIP-base for captioning and VQA, and Flan-T5-base for synthesis.
We resize input frames to $224\times224$ for CLIP and $384\times384$ for BLIP. Memory capacity $N\!=\!64$; TQR recent window $W\!=\!30$\,s.
All inference uses FP16 mixed precision via \texttt{torch.autocast}.
All results are reported on an NVIDIA A100-SXM4-40GB GPU with fixed random seeds (seed=42) for reproducibility.

\subsection{Main Results}

\paragraph{LiveQA-Bench.}
\Cref{tab:liveqa} reports \ours on LiveQA-Bench (55 streams) alongside reference baselines.
\ours reaches 54.8\% overall accuracy with 254\,ms average latency, with clear differentiation across scopes.

\noindent\textit{Historical} (77.1\%): The strongest scope. The full SKM provides deduplicated observations across the entire stream, and Flan-T5 synthesis effectively summarizes them. Importance-scored memory retains salient events from minutes earlier that FIFO and uniform sampling discard.

\noindent\textit{Instant} (46.2\%): Accuracy depends entirely on single-frame BLIP perception. Across 55 diverse visual domains---from cooking close-ups to sports action and cinematic sequences---BLIP-base sometimes fails to produce a caption that overlaps with the ground truth, despite describing the scene correctly in different words. TQR's routing to a single frame is critical: without it, instant accuracy drops to 30.9\% (\cref{tab:ablation}).

\noindent\textit{Recent} (43.2\%): Requires localizing the relevant context within a 30\,s window. This scope improves over instant because multiple keyframes provide redundant evidence, but the narrow window sometimes captures insufficient context in slow-paced streams.

\paragraph{Context: established benchmarks.}
To situate \ours in the broader VideoQA landscape, we reference published results under standard full-video evaluation.
On EgoSchema (5K questions, 3-minute clips), Dispider achieves 55.6\%, LLaVA-Next-Video 43.9\%, and Video-LLaVA 38.4\%~\cite{he2024dispider}.
These models benefit from observing the complete video; under a streaming protocol where only frames up to $t_q$ are available, accuracy drops substantially.
Published OVO-Bench~\cite{li2025ovobench} results confirm a large gap between offline models (Qwen2-VL-7B: 50.4\%, LLaVA-OneVision-7B: 52.7\%) and streaming models (Dispider: 41.8\%, Flash-VStream: 33.6\%), with backward tracing as the weakest category---the regime where importance-scored memory provides the largest gains.
\ours's scope-aware memory filtering is complementary to these architectures: it could reduce the frames passed to their answer stages, combining their stronger language models with our temporal routing.

\paragraph{Effect of SKM.}
Replacing SKM with a FIFO buffer reduces accuracy across all scopes: instant drops 1.3 points, recent 1.6 points, and historical 0.7 points, yielding 53.6\% overall ($-$1.2).
The historical gap (76.4\% vs.\ 77.1\%) confirms that importance scoring retains distinctive moments that recency-based eviction overwrites.

\paragraph{Effect of TQR.}
Disabling TQR drops overall accuracy from 54.8\% to 50.9\% ($-$3.9 points), validating scope-aware routing.
The per-scope breakdown is revealing: instant accuracy collapses from 46.2\% to 30.9\% ($-$15.3 points) because the generator receives all stored frames and produces a historical summary instead of describing the current frame.
Recent accuracy \emph{rises} from 43.2\% to 52.6\% ($+$9.4 points) because it avoids occasional misrouting by the keyword heuristic.
On balance, TQR's benefit for instant queries outweighs its cost for recent queries.
Replacing the keyword heuristic with a learned classifier (\cref{sec:conclusion}) could preserve the instant gains while eliminating the recent misrouting.

\paragraph{Effect of memory size.}
The $N\!=\!16$ configuration achieves the highest overall accuracy (57.2\%) with 49.2\% instant, 45.3\% recent, and 78.7\% historical.
Fewer slots concentrate on the most distinctive frames and reduce noise passed to the generator.
For $N \ge 32$, accuracy plateaus at 54.0--55.0\% overall, with historical stable at 77.1--77.3\%.
We expect larger memory to matter on longer streams where early events must be retained over many minutes.

\subsection{Error Analysis}
\label{sec:error_analysis}

We manually inspected incorrect LiveQA-Bench predictions and identified three dominant failure modes:

\paragraph{Scope misclassification (39\% of errors).}
The keyword-based TQR routes queries to the wrong temporal window.
Queries phrased ambiguously (e.g.\ \emph{``What is happening with the person?''}) lack temporal cues and default to \textsc{historical}, flooding the generator with irrelevant frames.
This parallels the challenge NExT-QA~\cite{xiao2021nextqa} reports for temporal questions: models struggle when the temporal structure must be inferred from context rather than explicit keywords.

\paragraph{Sparse keyframe coverage (32\% of errors).}
When the recent window contains zero or one keyframe, the generator lacks visual evidence. This affects rapid-transition scenes (e.g.\ cooking close-ups) where the SKM novelty threshold is too strict for fast-paced egocentric video.
An adaptive insertion threshold based on event density could mitigate this failure.

\paragraph{Language generation limitations (29\% of errors).}
Flan-T5-base occasionally produces correct but overly generic answers (e.g.\ \emph{``a person''} instead of \emph{``a woman in a blue shirt''}).
Open-ended answer generation remains a fundamental challenge shared across VideoQA~\cite{xiao2021nextqa}: models that perform well on multi-choice tasks struggle to generate specific answers without candidate options.

\paragraph{Failure cases.}
We highlight two representative failures from \cref{sec:error_analysis}.
\emph{Scope misrouting:} The question \emph{``What can you see in the current frame?''} contains ``current'' but is classified as \textsc{historical} because the TQR matches ``see'' against historical cues.
The generator receives all 64 keyframes and produces a summary of the full stream instead of describing the present frame; a learned classifier would correctly identify this as \textsc{instant}.
\emph{Generic generation:} For the \textsc{recent} question \emph{``What was the person doing recently?''}, BLIP correctly captions the keyframes as ``a person cutting vegetables on a cutting board,'' but Flan-T5 simplifies the answer to ``cooking''---losing the specific details that would match the ground truth.
Replacing Flan-T5-base with an instruction-tuned LLM would likely preserve these details.
